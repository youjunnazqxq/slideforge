# 启动和脚手架

本文档记录 SlideForge 前端项目在基础搭建阶段需要理解的问题，包括什么是脚手架、为什么使用 Vite、项目如何启动，以及常见启动问题。

## 1. 什么是脚手架

脚手架可以理解为“项目初始化工具”。

它会帮我们自动生成一个前端项目的基础结构，例如：

```text
package.json
index.html
src/
vite.config.ts
tsconfig.json
```

如果不用脚手架，我们需要手动配置很多东西：

- Vue 如何运行。
- TypeScript 如何编译。
- 浏览器如何加载页面。
- 开发服务器如何启动。
- 代码如何打包。
- CSS 如何处理。

脚手架的作用就是先把这些基础配置搭好，让我们可以直接开始写页面。

## 2. 为什么使用 Vite

SlideForge 前端计划使用：

```text
Vue3 + TypeScript + Vite
```

其中 Vite 是前端开发和构建工具。

Vite 主要负责：

- 启动本地开发服务器。
- 支持 Vue 单文件组件。
- 支持 TypeScript。
- 支持热更新。
- 构建生产环境代码。

开发时我们运行：

```bash
npm run dev
```

Vite 会启动一个本地服务，例如：

```text
http://localhost:5173
```

然后我们就可以在浏览器里访问前端页面。

## 3. 推荐项目结构

因为 SlideForge 后续会有前端、后端、文档等多个部分，所以建议采用如下结构：

```text
SlideForge/
  docs/
    frontend/
      基础的搭建/
        启动和脚手架.md

  apps/
    web/
      package.json
      index.html
      vite.config.ts
      tsconfig.json
      src/
        main.ts
        App.vue
        router/
        stores/
        views/
        components/
```

其中：

- `docs/` 存放项目文档。
- `apps/web/` 存放 Vue3 前端项目。
- 后续如果增加后端，可以放在 `apps/api/`。

这样结构比较清晰：

```text
apps/web 负责前端
apps/api 负责后端
docs 负责文档
```

## 4. 创建 Vue3 项目的方式

后续真正开始搭建时，可以在项目根目录执行：

```bash
npm create vite@latest apps/web -- --template vue-ts
```

这条命令的含义：

```text
npm create vite@latest   使用最新版 Vite 创建项目
apps/web                 把项目创建到 apps/web 目录
--template vue-ts        使用 Vue3 + TypeScript 模板
```

创建完成后，进入前端目录：

```bash
cd apps/web
```

安装依赖：

```bash
npm install
```

启动开发服务器：

```bash
npm run dev
```

## 5. Windows PowerShell 下的 npm 问题

在 Windows PowerShell 中，有时直接运行：

```bash
npm --version
```

可能会遇到类似问题：

```text
无法加载文件 npm.ps1，因为在此系统上禁止运行脚本
```

这是 PowerShell 执行策略导致的，不一定是 Node 或 npm 没装好。

可以临时使用：

```bash
npm.cmd --version
```

或：

```bash
npm.cmd install
npm.cmd run dev
```

也就是说，在当前项目里如果遇到 `npm.ps1` 限制，可以优先使用 `npm.cmd`。

## 6. 前端启动流程

完整启动流程可以理解为：

```text
安装 Node.js
  ↓
使用 Vite 创建 Vue3 项目
  ↓
安装依赖
  ↓
运行 npm run dev
  ↓
浏览器访问本地地址
```

命令形式：

```bash
cd apps/web
npm install
npm run dev
```

如果在 PowerShell 里遇到执行策略问题，则使用：

```bash
cd apps/web
npm.cmd install
npm.cmd run dev
```

## 7. package.json 的作用

`package.json` 是前端项目的核心配置文件。

它主要记录：

- 项目名称。
- 项目版本。
- 运行脚本。
- 项目依赖。
- 开发依赖。

常见内容：

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vue-tsc -b && vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "vue": "^3.0.0"
  },
  "devDependencies": {
    "vite": "^5.0.0",
    "typescript": "^5.0.0"
  }
}
```

其中：

- `npm run dev` 用于本地开发。
- `npm run build` 用于打包生产版本。
- `npm run preview` 用于预览打包后的结果。

## 8. Tailwind CSS 的加入时机

创建 Vue3 项目后，再加入 Tailwind CSS。

Tailwind 的作用是提供一套工具类样式，例如：

```html
<button class="rounded-md bg-zinc-900 px-4 py-2 text-white">
  登录
</button>
```

它会用于：

- 登录页布局。
- App 主布局。
- 侧边栏。
- 顶部栏。
- 表单。
- 卡片。
- 工作台页面。

第一阶段不需要复杂主题系统，只需要先配置 Tailwind，让项目能正常使用工具类即可。

## 9. 基础依赖建议

前端第一阶段建议依赖：

```text
vue
vue-router
pinia
tailwindcss
lucide-vue-next
```

作用分别是：

- `vue`：前端框架。
- `vue-router`：页面路由。
- `pinia`：状态管理。
- `tailwindcss`：样式工具。
- `lucide-vue-next`：图标库。

第一阶段先不要加入太多库，避免项目还没开始就变复杂。

## 10. 第一阶段脚手架验收标准

前端脚手架搭好后，应满足：

```text
1. 可以执行 npm install。
2. 可以执行 npm run dev。
3. 浏览器可以访问本地页面。
4. 页面可以显示 Vue 应用。
5. Tailwind 样式可以生效。
6. Vue Router 可以切换页面。
7. Pinia 可以保存基础状态。
```

达到这些标准后，再开始编写登录页、Layout 和用户详情页。

## 11. 当前阶段注意事项

当前阶段只讨论和准备前端基础搭建，不急着接入：

- 后端 API。
- AI 调用。
- 数据库。
- 真实登录鉴权。
- PPT 生成流程。

先把前端项目跑起来，再逐步往里面加功能。
